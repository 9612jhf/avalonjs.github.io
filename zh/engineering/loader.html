AMD加载器,avalon自带强大的加载器

AMD(异步模块定义)是什么？

JavaScript模块只是遵循SRP(Single Responsibility Principle单一职责原则)的代码段， 它暴露了一个公开的API。在现今JavaScript开发中，你可以在模块中封装许多功能， 而且在大多数项目中，每个模块都有其自己的文件。这使得JavaScript开发者日子有点难过， 因为它们需要持续不断的关注模块之间的依赖性，按照一个特定的顺序加载这些模块，否则运行时将会放生错误。

当你要加载JavaScript模块时，就会使用script标签。为了加载依赖的模块，你就要先加载被依赖的，之后再加载依赖的。 使用script标签时，你需要按照此特定顺序安排它们的加载，而且脚本的加载是同步的。 可以使用async和defer关键字使得加载异步，但可能因此在加载过程中丢失加载的顺序。 另一个选择是将所有的脚本捆绑打包在一起，但在打包后你仍然需要把它们按照正确的顺序排序。

AMD就是这样一种对模块的定义，使模块和它的依赖可以被异步的加载，但又按照正确的顺序。

avalon内置了AMD加载器，用于与其原创者requirejs一模一样，但更轻量。它主要涉及以下三个函数:

  • define(id?, deps?, factory)– 一个全局函数，用于定义模块。
  • require(deps, factory)– 一个全局函数，用于加载N个模块，以后将它们作为参数依次传入第二个参数factory中。
  • require.config(settings)- 用于配置加载器的各种行为

此外， 我们还可以在define.amd对象查看到所有加载过的模块。

baseUrl配置项的使用

avalon1.4的新特性, 有时, javascript文件并不是放在根目录下,我们每个文件就需要写得长, 有了它, 每个可以写很短

在实际使用过程中，如果用户没有配置baseUrl，会默认使用avalon.js的目录当做baseUrl，但绝对不会使用base标签的href当做baseUrl，这是为了与requirejs保持一致。

例子, 见下面目录结构

主页面的index.html如下:

<!DOCTYPE html>
<html>
    <head lang="en">
        <meta charset="UTF-8">
        <title></title>
        <script type="text/javascript" src="avalon.js"></script>
        <script type="text/javascript">
            require.config({
                baseUrl: "./examples/loader/"
            })
            require(["./aaa"], function(a){
                avalon.define({
                    $id: "test",
                    title: "加载成功"
                })
                avalon.log(a)
                avalon.scan()
            })
        </script>
    </head>
    <body>
        <div ms-controller="test">
            <h1>{{title}}</h1>
        </div>
    </body>
</html>

由于我们设置baseUrl, 当我们加载./aaa模块时, 它的路径就变成./examples/loader/aaa, 再自动加上.js后缀名

我们再看一下所有JS文件是如何写的:

aaa.js

define(["./bbb"], function(){
     avalon.log("已经加载aaa模块")
    return "aaa"
})

bbb.js

    avalon.log("已经加载bbb模块")
    return "bbb"
})

ccc.js

    avalon.log("已经加载ccc模块")
    return "ccc"
})

ddd.js

define(["./eee"], function() {
    avalon.log("已经加载ddd模块")
    return "ddd"
})

eee.js

    avalon.log("已经加载eee模块")
    return "eee"
})

paths配置项的使用

可以让一些非常长的路径名改成一个单词，然后我们直接在require中引用这些单词就行了。

看下面例子， 这几个例子内部都使用了amd机制，支持三种引入机制。

jquery.placeholder可以在这里下，https://github.com/mathiasbynens/jquery-placeholder

主页面的index.html如下:

<!DOCTYPE html>
<html>
    <head lang="en">
        <meta charset="UTF-8">
        <title></title>
        <script type="text/javascript" src="avalon.js"></script>
        <script type="text/javascript">
            require.config({
                baseUrl: "./lib/",
                paths: {
                    jquery: "jquery-2.1.3",
                    "jquery.placeholder": "jquery.placeholder",
                    underscore: "underscore-1.7",
                    backbone: "backbone"
                }
            })
            require(["backbone", "jquery.placeholder"], function(a) {
                console.log(a, jQuery.fn.placeholder)
                avalon.define({
                    $id: "test",
                    title: "加载成功"
                })
                avalon.scan()
            })
        </script>
    </head>
    <body>
        <div ms-controller="test">
            <h1>{{title}}</h1>
        </div>
    </body>
</html>

shim配置项的使用

shim是用于将加载那些不符合AMD规范的JS文件, 它拥有两种形态, 数组形态(相当于只配置了deps参数), 对象形态(可配置deps, exports, init这三个参数)

avalon1.4才完完整整支持此功能。

requirejs.config({
    shim: {
        'backbone': {
            //指定依赖项,必须为数组, 不写时默认为空数组,它们会先于被依赖者加载
            deps: ['underscore', 'jquery'],
            exports: 'Backbone'//指定它的命名空间
        },
        'underscore': {
            exports: '_' //指定它的命名空间
        },
        'jquery.selectBoxIt': ['jquery'],//指定它的依赖项
        'backbone.Validation': ['backbone'],//指定它的依赖项
        "jquery.placeholder": ["jquery"],//指定它的依赖项
        'foo': {
            deps: ['bar'],
            exports: 'Foo',
            init: function (bar) {
                //Using a function allows you to call noConflict for
                //libraries that support it, and do other cleanup.
                //However, plugins for those libraries may still want
                //a global. "this" for the function will be the global
                //object. The dependencies will be passed in as
                //function arguments. If this function returns a value,
                //then that value is used as the module export value
                //instead of the object found via the 'exports' string.
                return this.Foo.noConflict();
            }
        }
    }

主页面的index.html如下:

<!DOCTYPE html>
<html>
    <head lang="en">
        <meta charset="UTF-8">
        <title>require shim机制</title>
        <script src="js/lib/avalon.js"></script>
        <script src="js/app.js"></script>
    </head>
    <body>
        <h2>通过shim机制加载那些没有使用AMD风格定义的jquery插件</h2>
        <p>如果加载成功一会儿这里下面会多出两行</p>
    </body>
</html>

js/app.js

// Place third party dependencies in the lib folder
// Configure loading modules from the lib directory,
// except 'app' ones, 
require.config({
    "baseUrl": "js/lib",
    "paths": {
      "app": "../app"
    },
    "shim": {
        "jquery.alpha": {
           deps: ["jquery"],
           init: function($){
               console.log($) //这里只是测试init配置项是否生效
           }
        },
        "jquery.beta": ["jquery"]
    }
});

// Load the main app module to start the app
require(["app/main"]);

js/app/main.js

define(["jquery", "jquery.alpha", "jquery.beta"], function($) {
    //the jquery.alpha.js and jquery.beta.js plugins have been loaded.
    $(function() {
        $('body').alpha().beta();
    });
});

js/lib/jquery.alpha.js

$.fn.alpha = function() {
    return this.append('
Alpha is Go!
');
};

js/lib/jquery.beta.js

$.fn.beta = function() {
    return this.append('
Beta is Go!
');
};

urlArgs配置项的使用

拥有两个传参方式,一种是传入一个字符串,那么所有要加载的请求都会加上它们,另一种是传入一个对象,让特殊模块才加上指定参数

该配置项可以用于处理缓存

require.config({
   urlArgs: "v="+ Math.random() 
})
//相当于
require.config({
   urlArgs: {
      "*" :"v="+ Math.random() 
   }
})
require.config({
   urlArgs: {
      "jquery" :"username=xxxx&pw=345354" ,
      "backbone": "username=ewre&pw=356688" ,
   }
})

主页面的index.html如下:

<!DOCTYPE html>
<html>
    <head lang="en">
        <meta charset="UTF-8">
        <title>urlArgs</title>
        <script src="avalon.js"></script>
        <script>
            require.config({
                baseUrl: "./jQuery/",
                urlArgs: {
                    "*": "v=" + (new Date - 0),
                    "jquery": "aaa=333"
                },
                "shim": {
                    "jquery.alpha": ["jquery"],
                    "jquery.beta": ["jquery"]
                }
            })
            require(["jquery.alpha", "jquery.beta"], function() {
                $(function() {
                    $('body').alpha().beta();
                });
            })

        </script>
    </head>
    <body>
        <h2>使用urlArgs添加参数</h2>
        <p>如果加载成功一会儿这里下面会多出两行</p>
    </body>
</html>

map配置项的使用

map参数用来解决同一个模块的不同版本问题，这一灵感来自于Dojo的packageMap。有这样的场景： 开发初期使用了的jquery-1.6.4，后期升级到了1.7.2。但担心有些依赖jquery-1.6.4的代码升级到1.7.2后有问题。 因此保守的让这部分代码继续使用1.6.4版本。这时map参数将派上用场。比如我们在new/aaa/目录使用jquery1.6.4，在old/aaa/项目下使用jquery1.7.4。

requirejs.config({
    map: {
        "old/aaa": {
            'jquery': 'jquery-1.6.4'
        },
        'new/aaa': {
            'jquery': 'jquery-1.7.2'
        }
    }
})
